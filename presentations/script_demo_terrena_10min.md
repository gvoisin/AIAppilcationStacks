# Script oral - Demo Terrena 10 minutes

## Introduction
Bonjour a tous.
Aujourd hui, je vais vous montrer une meme logique metier Terrena exposee sous trois formes applicatives tres differentes.
L objectif est de faire toucher du doigt la difference entre une application traditionnelle, un chatbot LLM, et une application agentique moderne.
Nous avons adapte le demonstrateur a des enjeux Terrena concrets: filieres, logistique, qualite, tracabilite, campagnes et marques comme La Nouvelle Agriculture, Pere Dodu, Douce France ou Ackerman.

## Partie 1 - Application traditionnelle
Je commence par la version traditionnelle.
Ici, nous avons une application de pilotage classique, avec des ecrans prepares a l avance.
On retrouve des indicateurs metier qui parlent a Terrena:
- la collecte La Nouvelle Agriculture
- les alertes ouvertes
- le service logistique
- la conformite marques et tracabilite

On voit aussi des alertes operationnelles sur le terrain.
Par exemple:
- un retard de collecte a Ancenis sur des flux La Nouvelle Agriculture
- un controle sanitaire renforce en Vendee avicole qui peut impacter Pere Dodu et Gastronome Professionnels
- un suivi documentaire export sur Ackerman
- une maintenance planifiee sur une ligne liee a Sourires de campagne

Cette application est tres utile pour piloter, mais elle oblige l utilisateur a naviguer dans les onglets, lire les tableaux et reconstituer lui-meme l histoire.

## Transition
La question suivante est simple:
que se passe-t-il quand on ne veut plus naviguer, mais simplement demander ?

## Partie 2 - Chatbot LLM
Dans la deuxieme etape, on garde les memes donnees, mais on change l interaction.
Au lieu d explorer des ecrans, l utilisateur peut poser des questions metier en langage naturel.
Par exemple:
- Quels incidents pourraient impacter Pere Dodu cette semaine ?
- Y a t il un risque pour les expeditions Ackerman ?
- Quels sujets doivent etre remontes a un responsable qualite pour Douce France ?

Le benefice est immediat.
Le systeme synthese, priorise et reformule l information.
On gagne du temps.
On rend la donnee plus accessible a un responsable de filiere, a un coordinateur logistique ou a un manager qualite.
Mais le systeme reste encore centré sur la reponse.
Il explique.
Il n organise pas encore l action.

## Transition
C est la qu arrive la troisieme etape.

## Partie 3 - Application agentique
Dans la version agentique, le systeme ne repond plus seulement a une question.
Il comprend une intention metier et reconstruit dynamiquement une experience adaptee.

Par exemple, si je demande:
Construis un espace de pilotage pour un risque sanitaire sur la filiere volailles avec impacts potentiels sur Pere Dodu et Gastronome Professionnels.

Le systeme peut assembler une vue de crise avec:
- les alertes prioritaires
- les sites et territoires concernes
- les impacts logistiques
- les points de controle qualite
- les prochaines actions a suivre

Autre exemple:
Prepare un cockpit de suivi des expeditions vins pour Ackerman avec alertes de tracabilite, lots a risque et priorites de traitement.

Ici, l application se configure autour du besoin du moment.
On n est plus dans une logique d ecran unique ou de simple reponse.
On est dans une logique de composition dynamique d un outil de travail.

## Message cle pour les developpeurs
Pour les developpeurs, la difference est tres nette.
Dans le monde traditionnel, on code les ecrans a l avance.
Dans le monde LLM, on ajoute une couche conversationnelle sur les donnees.
Dans le monde agentique, on construit un systeme capable de choisir, assembler et adapter lui-meme l experience selon le contexte metier.

## Conclusion
Si je resume en une phrase:
- le traditionnel permet de naviguer dans des ecrans fixes
- le LLM permet de dialoguer avec les donnees
- l agentique permet d orchestrer dynamiquement une action metier

C est exactement ce que cette demonstration Terrena cherche a rendre visible.
Sur un meme socle de donnees, on change profondement la valeur produite pour les utilisateurs.
